ÉTAPE 1 — MAPPING ANCIENNES → NOUVELLES COLONNES  [TERMINÉE]
────────────────────────────────────────────────────────────────────────────────
^
COLONNES SUPPRIMÉES (absentes de la nouvelle structure)
  - Identifiant de document
  - Reference document
  - 1/2/3/4/5 Articles CGI
  - Identifiant local
  - Prefixe de section   (encodé dans id_parcelle)
  - Section              (encodé dans id_parcelle, chars 9-10)
  - No plan              (encodé dans id_parcelle, chars 11-14)

COLONNES AJOUTÉES
  - id_mutation          : identifiant de mutation (groupe les lignes d'une même vente)
  - id_parcelle          : identifiant cadastral 14 chars (ex: "75104000AJ0071")
  - ancien_id_parcelle
  - ancien_code_commune
  - ancien_nom_commune
  - longitude            : DECIMAL(10,7), WGS-84
  - latitude             : DECIMAL(10,7), WGS-84

MAPPING COLONNES (ancienne → nouvelle)
  No disposition              → numero_disposition
  Date mutation (DD/MM/YYYY)  → date_mutation (YYYY-MM-DD ISO)  ← format changé
  Nature mutation             → nature_mutation
  Valeur fonciere (varchar,)  → valeur_fonciere (DECIMAL, .)    ← type changé
  No voie                     → adresse_numero
  B/T/Q                       → adresse_suffixe
  Type de voie + Voie         → adresse_nom_voie                ← FUSION (ex: "RUE DE RIVOLI")
  Code voie                   → adresse_code_voie               (valeur identique ✓)
  Code postal (INT)           → code_postal (VARCHAR 5)         ← type changé
  Commune ("PARIS 01")        → nom_commune ("Paris 1er...")    ← format changé
  Code departement            → code_departement
  Code commune (INT 3)        → code_commune (VARCHAR 5)        ← ex: "75"+"104" = "75104"
  No Volume                   → numero_volume
  1er lot … 5eme lot          → lot1_numero … lot5_numero
  Surface Carrez du 1er lot … → lot1_surface_carrez …          ← type changé (DECIMAL)
  Nombre de lots              → nombre_lots
  Code type local             → code_type_local
  Type local                  → type_local                       (identique ✓)
  Surface reelle bati         → surface_reelle_bati             ← type changé (DECIMAL)
  Nombre pieces principales   → nombre_pieces_principales
  Nature culture              → nature_culture
  Nature culture speciale     → nature_culture_speciale
  Surface terrain             → surface_terrain

POINTS CLÉS RELEVÉS SUR LES DONNÉES PARIS (fichier 2025)
  - nom_commune      = "Paris 4e Arrondissement" (accentué, lisible)
  - code_commune     = "75104" (INSEE, distingue chaque arrondissement)
  - adresse_code_voie = même valeur FANTOIR 4 chars qu'avant → continuité assurée
  - lot1_surface_carrez = "93.54" (point décimal, plus de REPLACE nécessaire)
  - date_mutation    = "2025-01-03" (ISO, plus de STR_TO_DATE nécessaire)
  - id_parcelle      = "75104000AJ0071"
      → section = SUBSTRING(id_parcelle, 9, 2) = "AJ"
      → no_plan = SUBSTRING(id_parcelle, 11, 4) = "0071"
